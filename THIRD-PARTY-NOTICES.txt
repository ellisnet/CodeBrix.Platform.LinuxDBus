CodeBrix.Platform.LinuxDBus is a low-level D-Bus protocol library for
Linux, ported from the upstream `Tmds.DBus.Protocol` NuGet package
(version 0.21.3) to a .NET 10 target and the CodeBrix family's coding
conventions. Substantially all of the source code (and all of the test
code) in this repository is derived from that upstream project and is
redistributed here under its original MIT License.

Jeremy Ellis and contributors license the CodeBrix.Platform.LinuxDBus
project to you under the terms set forth in the LICENSE file, which is
located at the root of this repository. The MIT License of the upstream
Tmds.DBus project and this repository's own MIT License are identical
in terms; the upstream copyright and permission notice is reproduced
below in full, as required by that license.

If you have any questions about any third-party code or services
referenced here, or if you believe that any third-party code has been

    Security:          This port is taken from the rel/0.21.3 tag,
                       which includes the fixes for advisory
                       GHSA-xrw6-gwf8-vvr9 / CVE-2026-39959 (High,
                       CVSS 7.1). The three fixed attack vectors are:
                       signal-sender spoofing (malicious peers
                       impersonating well-known names), Unix file
                       descriptor resource exhaustion / fd spillover,
                       and denial-of-service via unhandled exceptions
                       on the SynchronizationContext inside AddMatch
                       callbacks.

    Every .cs file in src/CodeBrix.Platform.LinuxDBus/ and in
    tests/CodeBrix.Platform.LinuxDBus.Tests/ is derived from its
    counterpart in src/Tmds.DBus.Protocol/ or
    test/Tmds.DBus.Protocol.Tests/ of the upstream repository at the
    rel/0.21.3 tag. Each ported file carries a
    `//was previously: Tmds.DBus.Protocol[.*];` provenance comment on
    its `namespace` line.

    Modifications made during the port:
    * Renamed the root namespace from `Tmds.DBus.Protocol` to
      `CodeBrix.Platform.LinuxDBus` and split the source into sub-
      namespaces (Messages, Reading, Writing, Types, Exceptions,
      Enumerations, Internal).
    * Narrowed the target framework from
      `netstandard2.0;netstandard2.1;net6.0;net8.0` to `net10.0` only,
      and stripped every `#if NETSTANDARD2_0 / NETSTANDARD2_1 / NET6_0 /
      NET7_0 / NETFRAMEWORK` conditional-compile branch that is always
      false on .NET 10. Live branches were kept verbatim; the removed
      branches included the entire `Netstandard2_0Extensions.cs`,
      `Netstandard2_1Extensions.cs`, and the netstandard-only portions
      of the `Polyfill/` folder.
    * Deleted `GlobalUsings.cs` and promoted every global `using` to an
      explicit file-scoped `using` in the files that need it (the
      CodeBrix family forbids global usings).
    * Removed `<Nullable>enable</Nullable>` and stripped every `?`
      annotation on reference types and every null-forgiveness `!`
      operator (the CodeBrix family has nullable reference types off).
    * Ported tests from xUnit (v2) to xUnit v3 and replaced
      `Assert.*(...)` calls with SilverAssertions fluent
      `.Should()...` calls where practical. Added
      `TestContext.Current.CancellationToken` on cancellable awaits.
    * Added XML doc comments on every public and protected member so
      that `<GenerateDocumentationFile>true</GenerateDocumentationFile>`
      compiles cleanly (CS1591-free).
    * Removed the `SignAssembly` / `AssemblyOriginatorKeyFile` /
      `PublicSign` properties and the embedded `sign.snk` resource;
      this port is not strong-name signed.
